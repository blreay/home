# Sleepstudy 实锤分析 —— 昨晚发烫元凶定位

> 数据来源：`d:\sleepstudy.html`（`powercfg /sleepstudy`，含过去 3 天现代待机记录）
> 分析时间：2026-06-17
> 目标会话：2026-06-16 21:28:00 起、时长 1:52:21 的那次待机

---

## 一、一句话结论

**元凶是「英特尔® 智音技术总线（Intel Smart Sound Technology / HDAS）」——音频总线。** 在昨晚 1 小时 52 分的待机里，它有 **1 小时 17 分（69%）一直处于活跃状态没有休眠**，Windows 直接把它标记为 **`Problem Device: TRUE`（问题设备）**。这导致整机全程无法进入深度低功耗（DRIPS 转换次数 = 0），持续 ~8.7W 发热掉电。

> 通俗说：你盖盖子时，**有程序占着声卡（音频流）没放**，声卡驱动就一直"醒着"，把整台电脑拖住睡不沉。

## 三、根因链条

```
盖盖子前有程序占用音频流（疑似 Antding 提示音 / 浏览器媒体标签 / 通知音）
        ↓
进入现代待机后，该音频流没有释放
        ↓
Intel Smart Sound 音频总线 (HDAS) 无法进入低功耗，全程 69% 时间活跃
        ↓
整机无法进入 DRIPS 深度低功耗（转换次数 = 0）
        ↓
CPU/平台维持 ~8.7W → 持续发热 + 1小时52分掉34%电
        ↓
（休眠被禁用，无兜底）→ 电池被一路榨干
```

## 四、针对性解决方案（按优先级）

### ⭐ 1. 装上「兜底休眠」（最重要，立即做）

即使下次音频又把待机拖住，30 分钟后强制休眠断电，从此不会再被烤干。
→ **直接运行之前给你的 `修复盖盖子发烫.bat`（管理员）即可。**

### ⭐ 2. 盖盖子前先「掐掉音频源」（针对本次元凶）

盖盖子前养成习惯：
- 退出 / 静音 **Antding（钉钉）**、会议软件、音乐播放器；
- 关掉浏览器里**正在播放或可能自动播放**的媒体标签页；
- 暂停一切正在播放的视频/音频。

> 验证方法：盖盖子前看任务栏右下角喇叭图标，确认没有"正在播放"的小动画；或在「设置 → 系统 → 声音 → 音量合成器」里确认没有 App 在出声。

### 3. 禁止音频设备维持/唤醒系统（管理员，可选）

```powershell
:: 查看当前可唤醒设备
powercfg /devicequery wake_armed

:: 如音频设备在列，禁用其唤醒能力（按实际名称替换）
powercfg /devicedisablewake "Realtek(R) Audio"
```

> 注意：这只防"唤醒"，对"占流不释放"作用有限，主力还是方案 1+2。

### 4. 更新 Intel Smart Sound (SST) 音频驱动（治本）

`Problem Device: TRUE` 常由 SST 驱动 bug 引起。到**笔记本厂商官网**下载最新的：
- **Intel Smart Sound Technology (SST) 驱动**
- **声卡（Realtek）驱动**
- 顺带更新 **BIOS / 芯片组驱动**

这是从根上减少"音频赖着不睡"的最有效手段。

### 5. 排查 Antding 等 IM 的后台音频保活

若怀疑是钉钉/Antding：在其设置里**关闭消息提示音 / 后台运行 / 开机自启**，或盖盖子前彻底退出（托盘右键退出，而非只关窗口）。

## 附：本次分析所用关键字段说明

| 字段 | 含义 |
|------|------|
| **% LOW POWER STATE TIME (SW/HW)** | 待机期间处于低功耗的时间占比，越高越好（>90% 健康） |
| **DRIPS / DripsTransitions** | 进入"最深低功耗"的次数，0 = 整夜没真正深睡 |
| **Problem Device: TRUE** | Windows 判定该设备是拖住待机的问题设备 |
| **Fx Device + 高 ACTIVE TIME** | 该硬件设备在待机里长时间没进低功耗 |
| **CHANGE RATE (mW)** | 平均功耗，健康待机应 <500mW，昨晚 8751mW |
